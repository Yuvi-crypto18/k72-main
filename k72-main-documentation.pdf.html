1. Project Overview

📝 Description

K72-Main is a modern, animated portfolio website built for a creative digital agency. The project showcases advanced web development techniques including smooth page transitions, scroll-based animations, and a sophisticated full-screen navigation system.

🎯 Project Type

Portfolio Website Single Page Application Animation Showcase

✨ Key Features

  • 🎬 Full-screen video background with embedded video elements
  • 🔄 Smooth page transitions with "stairs" animation effect
  • 📱 Fully responsive design for all device sizes
  • 🎨 Custom navigation with hover effects and animations
  • 📜 Scroll-triggered animations using GSAP ScrollTrigger
  • 🖼️ Dynamic image galleries with hover effects
  • 🌐 Multi-language support (French content)
  • ⚡ Optimized performance with Vite build tool

5. Algorithms & Logic

🧮 Core Algorithms

1. Stairs Transition Algorithm

Creates a sequential "stairs" animation effect during page transitions:

Algorithm: Staggered Vertical Reveal
─────────────────────────────────────
1. Divide screen into 5 equal vertical sections
2. Animate sections from right to left:
   - Section 5 appears first (height: 0 → 100%)
   - Section 4 appears with 0.04s delay
   - Section 3 appears with 0.08s delay
   - Section 2 appears with 0.12s delay
   - Section 1 appears last (0.16s delay)
3. Once all visible, slide up simultaneously:
   - All sections move y: 0 → y: 100% (staggered)
4. Reset positions while hidden for next transition

2. Scroll-Based Image Sequence

On the Agence page, scrolling changes team member images:

Algorithm: Scroll-Progress Image Swapper
──────────────────────────────────────
Input: 14 team member images, scroll progress (0-1)

function onScrollUpdate(progress):
    if progress < 1:
        imageIndex = floor(progress × imageArray.length)
    else:
        imageIndex = imageArray.length - 1
    
    imageRef.current.src = imageArray[imageIndex]

Calculation Example:
- Scroll Progress: 0.25 → Image Index: floor(0.25 × 14) = 3
- Scroll Progress: 0.50 → Image Index: floor(0.50 × 14) = 7
- Scroll Progress: 0.75 → Image Index: floor(0.75 × 14) = 10
- Scroll Progress: 1.00 → Image Index: 13 (last image)

3. Staggered Project Card Animation

Projects reveal as you scroll:

Algorithm: ScrollTrigger Height Animation
────────────────────────────────────────
Initial State: height = 100px (collapsed)
Final State:   height = 850px (full height)

ScrollTrigger Configuration:
- Trigger: When .lol element enters viewport
- Start: When top of trigger is at 100% of viewport
- End: When top of trigger is at -150% of viewport
- Scrub: true (animation tied to scroll position)
- Stagger: 0.4s between each card

Visual Flow:
Scroll ↓
Card 1: 100px → 850px
   ↓ (0.4s delay)
Card 2: 100px → 850px
   ↓ (0.4s delay)
Card 3: 100px → 850px

4. Navigation Color Logic

Navbar adapts its color based on the current page:

Algorithm: Route-Based Color Switcher
────────────────────────────────────
useEffect(() => {
    const currentPath = location.pathname
    
    if (currentPath === '/projects' || currentPath === '/agence') {
        setNavColor('black')     // Dark pages need black logo
    } else {
        setNavColor('white')     // Light pages need white logo
    }

5. Full-Screen Navigation Timeline

Complex multi-step animation sequence:

Opening Animation Sequence:
──────────────────────────
Step 1: Display fullscreen overlay (display: block)
Step 2: Reveal stairs (height: 0 → 100%, staggered -0.3s)
Step 3: Show navigation links (opacity: 0 → 1, rotateX: 90 → 0)
Step 4: Fade in close button (opacity: 0 → 1)

Closing Animation Sequence:
──────────────────────────
Step 1: Hide navigation links (opacity: 1 → 0, rotateX: 0 → 90)
Step 2: Collapse stairs (height: 100% → 0, staggered 0.1s)
Step 3: Fade out close button (opacity: 1 → 0)
Step 4: Hide fullscreen overlay (display: none)

7. Animation Tools & Techniques

🎬 Animation Library: GSAP (GreenSock Animation Platform)

GSAP is a professional-grade JavaScript animation library used throughout the project.

📚 GSAP Modules Used

Module Purpose Usage
gsap.timeline() Sequential animations Navigation menus, page transitions
gsap.from() Animate from state Stairs effect, element reveals
gsap.to() Animate to state Navigation close, scrolling effects
ScrollTrigger Scroll-based animations Project cards, image sequences
useGSAP React integration Cleanup and lifecycle management

🔄 Animation Types Implemented

1. Page Transition Animation (Stairs)

gsap.timeline()
  .to(stairParent, { display: 'block' })
  .from('.stair', {
      height: 0,
      stagger: { amount: -0.2 }  // Right-to-left reveal
  })
  .to('.stair', {
      y: '100%',
      stagger: { amount: -0.25 }  // Slide up
  })
  .to(stairParent, { display: 'none' })
  .to('.stair', { y: '0%' })      // Reset for next use

2. ScrollTrigger Animation

gsap.registerPlugin(ScrollTrigger)

gsap.from('.hero', {
  height: '100px',
  stagger: { amount: 0.4 },
  scrollTrigger: {
    trigger: '.lol',
    start: 'top 100%',      // When trigger enters viewport
    end: 'top -150%',       // Animation end point
    scrub: true,            // Tie animation to scroll
  }
})

3. Image Sequence Animation

gsap.to(imageDiv, {
  scrollTrigger: {
    trigger: imageDiv,
    start: 'top 28%',
    end: 'top -70%',
    pin: true,              // Pin element during scroll
    scrub: 1,               // Smooth scrolling
    onUpdate: (self) => {
      // Calculate image index based on scroll progress
      const index = Math.floor(self.progress * images.length)
      imageRef.src = images[index]
    }
  }
})

💫 CSS Animations

/* Infinite horizontal scroll */
@keyframes moveAnimation {
    from { transform: translateX(0); }
    to { transform: translateX(-100%); }
}

.moveX {

/* Hover fade effect */

⚙️ Animation Configuration

Property Value Effect
stagger -0.2s to 0.4s Delay between animated elements
scrub true or 1 Tie animation to scroll position
pin true Fix element during scroll animation
duration 10s CSS animation duration

Key Code Patterns

1. GSAP Integration Pattern

import { useGSAP } from '@gsap/react'
import gsap from 'gsap'

useGSAP(function () {
  // Animation code here
  gsap.to('.element', { /* properties */ })
}, [dependencies])

Benefits:
- Automatic cleanup on unmount
- Runs when dependencies change
- React-friendly lifecycle

2. Context Usage Pattern

// Create contexts
export const NavbarContext = createContext()
export const NavbarColorContext = createContext()

// Provider component
const NavContext = ({ children }) => {
  const [navOpen, setNavOpen] = useState(false)
  const [navColor, setNavColor] = useState('white')
  
  return (
    <NavbarContext.Provider value={[navOpen, setNavOpen]}>
      <NavbarColorContext.Provider value={[navColor, setNavColor]}>
        {children}
      </NavbarColorContext.Provider>
    </NavbarContext.Provider>
  )
}

// Consumer component
const [navOpen, setNavOpen] = useContext(NavbarContext)

3. ScrollTrigger Pattern

import { ScrollTrigger } from 'gsap/all'

gsap.registerPlugin(ScrollTrigger)

gsap.from('.element', {
  // Initial state
  height: '100px',
  
  scrollTrigger: {
    trigger: '.trigger-element',
    start: 'top 100%',    // When trigger top hits viewport bottom
    end: 'top -150%',     // When trigger top is 150% above viewport
    scrub: true,          // Smooth scroll-linked animation
  }
})

4. Responsive Design Pattern

Tailwind Responsive Classes:
- Base (mobile): text-[12vw]

Pattern:
className='text-[12vw] lg:text-[9.5vw]'
          ↑ mobile     ↑ desktop

Common responsive pairs:
- p-2 lg:p-5 (padding)

5. Ref Pattern for GSAP

const elementRef = useRef(null)

useGSAP(() => {
  gsap.to(elementRef.current, {
    opacity: 1,
    duration: 1
  })
})

return <div ref={elementRef}>Content</div>

Why refs?
- Direct DOM access needed for GSAP
- Avoids React re-renders during animation
- Better performance for complex animations

🏆 Best Practices Implemented

  • Code Organization: Clear folder structure separating concerns
  • Animation Cleanup: useGSAP hook ensures animations are properly cleaned up
  • Accessibility: Semantic HTML structure
